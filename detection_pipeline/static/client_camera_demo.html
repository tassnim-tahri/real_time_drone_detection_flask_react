Live detection — this device’s camera

Frames are captured on whatever device opened this page (your phone’s camera when you browse on your phone, laptop webcam when on the laptop). They are sent to the server only for inference; frames are not stored by this demo. Server webcam stream (MJPEG) is unchanged.

Phones send each frame to your laptop over Wi‑Fi/TLS — that is inherently slower than the laptop webcam on this page. If it feels sluggish, leave Infer rate at Very slow (~1/sec) or pick a slower CPU model on the server (env CLIENT_FRAME_IMGSZ=320).

Laptop works but phone says «This site can't be reached / took too long»?
Almost always Windows Firewall blocking inbound TCP 8000, or Wi‑Fi AP / client isolation on the router. On the Flask PC run scripts/allow_firewall_port.ps1 as Administrator. Use the laptop’s IPv4 address (from ipconfig), not localhost. Phone and laptop must be on the same network (avoid guest Wi‑Fi).
HTTPS + camera: plain http://192.168.x.x usually disables getUserMedia. Use FLASK_DEV_HTTPS=1 then https://<laptop‑IP>:8000/... and trust the certificate.
